# 🛰 Грант-радар — Telegram-бот

Каждые несколько часов сам обходит фонды и грантовые Telegram-каналы, отбирает
**новые** конкурсы по твоим ключевым словам и присылает их в Telegram.
Работает бесплатно на GitHub Actions — **свой сервер не нужен**, ноутбук держать
включённым не надо.

Источники по умолчанию (Россия): РНФ, Фонд содействия инновациям (Фонд Бортника),
Grantfull, Капитан Грантов, ВсеКонкурсы. Легко меняются в `config.json`.

---

## 🚀 Как запустить (15 минут, без навыков программирования)

### 1. Создай бота и узнай токен
1. В Telegram открой **@BotFather** → `/newbot` → задай имя. 
2. Он пришлёт **токен** вида `1234567890:AAE...` — сохрани.

### 2. Узнай свой chat_id (куда слать)
1. Напиши что-нибудь своему новому боту (любое сообщение).
2. Открой в браузере (подставь токен):
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. Найди в ответе `"chat":{"id":123456789` — это твой **chat_id**.
   *(Хочешь получать гранты в группу — добавь бота в группу и возьми её id,
   он будет с минусом.)*

### 3. Залей проект на GitHub
1. Создай новый репозиторий на github.com.
2. Загрузи в него все файлы этой папки (кнопка **Add file → Upload files**
   или через git).

### 4. Добавь секреты
В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.
Создай два секрета:
- `TELEGRAM_BOT_TOKEN` — токен из шага 1
- `TELEGRAM_CHAT_ID` — id из шага 2

### 5. Включи и проверь
1. Вкладка **Actions** → если попросит, нажми «I understand… enable workflows».
2. Слева выбери **grant-radar** → **Run workflow** (ручной запуск для теста).
3. Через минуту в Telegram придёт «🛰 Грант-радар запущен!».
   Дальше он сам будет работать по расписанию (каждые 6 часов).

## ⚙️ Настройка под себя (`config.json`)

- **`keywords`** — темы фильтра. Пиши корни слов в нижнем регистре
  (`"физик"` поймает «физика», «физический», «физике»). Убери лишнее / добавь своё.
- **`sources`** — что обходить. Два типа:
  - `"type": "html"` — обычная статичная страница (диф по заголовкам).
  - `"type": "telegram"` — канал через `https://t.me/s/ИМЯ_КАНАЛА`.
    Добавить любой канал: скопируй строку и поменяй `url`.
- **`require_signal_word`** — если `true`, помимо ключевого слова в тексте
  должно быть «грант/конкурс/заявка…» (меньше мусора). Поставь `false`, если
  хочешь ловить шире.
- **`mode`** — как присылать находки:
  - `"stream"` — каждый новый грант приходит сразу, как найдётся.
  - `"digest"` — находки копятся и приходят **одним дайджестом** раз в неделю.
- **Расписание** — в `.github/workflows/`:
  - `radar.yml` (`cron`, UTC) — как часто **обходить** источники (по умолчанию
    каждые 6 часов). В режиме `digest` этот проход только копит находки.
  - `digest.yml` (`cron`, UTC) — когда **присылать** недельный дайджест
    (по умолчанию понедельник 06:00 UTC). Работает только для `mode: digest`.

## 🔎 Поиск по истории грантов

Бот копит всё найденное в архив (`archive.json`) — с заголовком, ссылкой,
источником, датой первого появления и **направлениями** (сработавшими ключевыми
словами). Когда обращается новый клиент, можно сразу показать, какие гранты по
его теме уже встречались.

**Через бота (Telegram).** Пользователь пишет тему или жмёт **кнопку-направление**
(`🏭 Машиностроение`, `⛏ Горное дело`, `🧬 Био и медицина`…) — бот отвечает
**карточками** грантов с датой и ссылкой, листаются кнопкой «⬇️ Ещё».
Команды: `/start` (приветствие + цифры архива), `/fresh` (свежие), `/stats`
(сколько чего в архиве), `/help`.

### ⚡ Запуск бота — мгновенные ответы
Чтобы бот отвечал **за секунды**, он работает постоянным процессом (long-polling):
```bash
# 1) создай файл .env рядом (в git не попадёт):
#    TELEGRAM_BOT_TOKEN=...
#    TELEGRAM_CHAT_ID=...
# 2) запусти:
./run-bot.sh          # (или: ./.venv/bin/python radar.py --serve)
```
Пока процесс запущен, бот отвечает моментально. Останавливать — `Ctrl+C`.
Позже тот же `--serve` без изменений переносится на сервер/VPS для работы 24/7.

> ⚠️ Нельзя держать два опросчика одновременно. Поэтому расписание `bot.yml`
> (опрос через GitHub Actions) **отключено** — бот живёт как `--serve`.

**Локально (без Telegram).**
```bash
./.venv/bin/python radar.py --search "горн"      # поиск в консоль
./.venv/bin/python radar.py --report             # собрать archive.md (Ctrl+F)
```

### 📁 Свои базы грантов (импорт истории)
Архив можно **пополнить своими историческими базами** — CSV, Excel (.xlsx) или JSON:
```bash
./.venv/bin/python radar.py --import data/            # все файлы из папки
./.venv/bin/python radar.py --import моя_база.xlsx    # или один файл
```
Колонки распознаются по названиям автоматически, направления проставляются сами.
Подробно и шаблон — в [`data/README.md`](data/README.md). Так «архив с нуля»
превращается в базу за годы — бот сразу ищет по ней.

**Поиск показывает структурированные гранты.** По умолчанию
(`config.json → bot_structured_only: true`) в поиске бота участвуют только гранты
из загруженных баз (`--import`) — с направлением, размером и сроком. Новостной
обход источников остаётся для ежедневных алертов, но карточками в поиске не
засоряет. Пока баз не загружено — временно показывается всё. Чтобы искать по
всему сразу, поставь `bot_structured_only: false`.

> Что радар собирает сам — копится **вперёд, с момента запуска** (сайты отдают
> только актуальные страницы). Историю за прошлые годы добавляй импортом баз (выше).

## 🧠 Как устроено (коротко)
- `radar.py` — весь движок. Команды: `--serve` (бот постоянным процессом),
  `--import` (загрузка баз), `--flush` (дайджест), `--search`/`--report`
  (локально), без флага — обход источников.
- `run-bot.sh` — запуск бота (`--serve`) с токеном из `.env`.
- `seen.json` — память «что уже показывал», чтобы гранты не повторялись.
- `pending.json` — очередь находок для режима `digest`.
- `archive.json` — исторический архив (сканер + импортированные базы), по нему
  ищет бот. `bot_state.json` — offset Telegram (локальный, в git не коммитится).
- `data/` — сюда кладут свои базы грантов для `--import` (см. `data/README.md`).
- Многие сайты фондов рендерятся через JS или отдают 403 роботам, поэтому упор
  на статичные страницы-новости и Telegram-каналы — так надёжнее.

## 🛠 Локальный запуск (по желанию)
```bash
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
# без токена — печатает находки в консоль:
./.venv/bin/python radar.py
# с отправкой в Telegram:
TELEGRAM_BOT_TOKEN=... TELEGRAM_CHAT_ID=... ./.venv/bin/python radar.py
```
