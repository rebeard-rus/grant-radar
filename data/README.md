# 📁 Свои базы грантов — сюда

Здесь можно подгружать **исторические базы грантов** (свои таблицы за прошлые
годы). Бот вольёт их в общий архив и сразу будет искать по ним — вместе с тем,
что радар собирает сам.

## Как загрузить

1. Положи в эту папку файл(ы): **CSV**, **Excel (.xlsx)** или **JSON**.
2. Выполни в терминале (из папки проекта):
   ```bash
   ./.venv/bin/python radar.py --import data/
   ```
   (можно указать и один файл: `--import data/моя_база.xlsx`)
3. Готово — бот уже ищет по этим грантам. Если бот запущен (`--serve`),
   он подхватит новые данные **на лету**, перезапуск не нужен.

## Какие колонки понимает

Заголовки распознаются автоматически (рус/англ, регистр не важен). Достаточно
хотя бы **названия** — остальное по возможности:

| Смысл | Подходящие названия колонок |
|---|---|
| **Название** (обязательно) | Название, Наименование, Заголовок, Title, Name, Грант, Конкурс |
| Организация / фонд | Организация, Фонд, Источник, Оператор, Source |
| Дата | Дата, Год, Date, Year (форматы: 2019-03-15, 15.03.2019, 2019) |
| Ссылка | Ссылка, URL, Link, Сайт |
| Направление / тема | Направление, Категория, Тема, Область, Category |
| Сумма | Сумма, Размер, Бюджет, Финансирование, Amount |
| Дедлайн | Срок, Дедлайн, Приём до, Deadline |
| Описание | Описание, Аннотация, Description, Суть |

Пример готового файла — `example_grants.csv` рядом. Направления бот дополнительно
проставит сам по ключевым словам из `config.json`.

> CSV лучше сохранять в кодировке UTF-8. Разделитель — `;` или `,` (определяется
> автоматически). Excel-файлы читаются как есть (нужен `openpyxl` из requirements).
